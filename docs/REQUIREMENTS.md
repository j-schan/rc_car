# RC카 요구사항 사양서 (Requirements Specification)

- 프로젝트: `rc_car` (STM32 기반 블루투스 RC카 → 미니 SDV 플랫폼)
- 작성일: 2026-07-05
- 버전: Rev.A (점퍼선 프로토타입 기준)

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 목적 | 블루투스 조종 아커만 RC카. 이후 엔코더 PID → 라즈베리파이 → CAN → 다중노드(미니 SDV)로 단계 확장 |
| 차체 | Hiwonder 아커만 스티어링 섀시 (단일 레이어, 금속) 27.0×20.2×7.3cm, 0.9kg, 정격하중 5kg |
| 조향 방식 | 아커만 (전륜 서보 조향 + 후륜 구동) — 차동식 아님 |
| 최소 회전 반경 | 0.55 m (섀시 스펙) |

## 2. 시스템 사양

### 2.1 제어기 (MCU)
| 항목 | 값 |
|------|-----|
| 보드 | NUCLEO-F446RE (STM32F446RET6, Cortex-M4F 180MHz) |
| 시스템 클럭 | 84 MHz (HSI + PLL) |
| 개발환경 | STM32CubeMX 6.17 + STM32CubeIDE, HAL, FW_F4 V1.28.3 |

### 2.2 구동계 (Drive)
| 항목 | 값 |
|------|-----|
| 모터 | JGB37-520R30-12 DC 기어모터 ×2 (후륜) |
| 정격 전압 / 전류 | 12V / 0.36A |
| 스톨 전류 | 3.2A/개 (⚠ L298N 채널당 연속 ~2A — 스톨 지속 금지) |
| 감속비 / 속도 | 1:30 / 무부하 320rpm |
| 엔코더 | AB 2상 11라인, VCC 3.3~5V → 출력축 1회전 = 1320카운트 (Phase 2) |
| 드라이버 | L298N (ENA/ENB 점퍼캡 제거, 5V-EN 점퍼 유지) |
| 모터 PWM | TIM3, 1kHz, 듀티 0~999 |
| 예상 최고속도 | ~0.6 m/s @ 3S (L298N 강하 후 ~9V 인가) |

### 2.3 조향계 (Steering)
| 항목 | 값 |
|------|-----|
| 서보 | LD-1501MG 디지털 (금속기어) |
| 동작 전압 | 6.0~8.4V (⚠ 9V 이상 인가 금지) |
| 전류 | 무부하 100mA / 스톨 2.4~3A |
| 토크 | 13kg·cm@6V, 17kg·cm@7.4V |
| 제어 | TIM2 PWM 50Hz, 500~2500µs = 0~180° |
| SW 안전 범위 | 1100~1900µs (중립 1500) — 코드에서 클램프 |

### 2.4 통신 (Comms)
| 항목 | 값 |
|------|-----|
| 블루투스 | HC-05, USART1 9600bps 8N1, 수신 인터럽트 |
| 디버그 | USART2 115200 (ST-Link VCP) — 텔레메트리/echo |
| 예약 | USART3(라즈베리파이), CAN1 500kbps(PA11/12, SN65HVD230 필요) |

### 2.5 전원 (Power) — 듀얼팩
| 항목 | 값 |
|------|-----|
| 구동계 | 3S 18650 (삼성 30Q ×3) = 11.1V → L298N +12V 입력 |
| 조향계 | 2S 18650 (삼성 30Q ×2) = 7.4V → 서보 직결 |
| 보드 | 개발: USB / 단독주행: L298N 5V 출력 |
| 셀 | Samsung INR18650-30Q (3000mAh, 15A) — 정품 필수 |
| ⚠ 철칙 | **+극은 분리(11.1V/7.4V), −극(GND)은 전체 공통** |

## 3. 기능 요구사항

| ID | 요구사항 | 단계 |
|----|----------|:---:|
| FR-01 | 스마트폰 앱에서 F/B/L/R/S 문자로 전진·후진·좌·우·정지 | P1 |
| FR-02 | '0'~'9' 문자로 속도 10단계 조절 | P1 |
| FR-03 | 부팅 시 모터 정지 + 서보 중립 (안전 초기화) | P1 |
| FR-04 | 수신 문자를 USART2로 echo (디버그) | P1 |
| FR-05 | 통신 두절 시 일정 시간 후 자동 정지 (페일세이프) | P1.5 |
| FR-06 | 엔코더 기반 속도 측정 및 PID 정속 제어 | P2 |
| FR-07 | 초음파 장애물 감지 시 자동 정지 | P2 |
| FR-08 | 라즈베리파이에서 UART로 cmd_vel 수신 / odom 송신 | P3 |
| FR-09 | CAN 버스로 제어·텔레메트리 메시지 전환 | P4 |

## 4. 단계 로드맵
```
P1 기본 주행(BT) → P2 정밀(엔코더·센서) → P3 파이(ROS2·카메라)
→ P4 CAN 전환 → P5 다중노드 미니 SDV(웹/OTA)
```

## 5. 하드웨어 개정 계획
| Rev | 내용 |
|-----|------|
| Rev.A | 점퍼선 + 브레드보드 프로토타입 (현재) |
| Rev.B | Nucleo 쉴드 PCB (EasyEDA 설계, JLCPCB 제작, CAN 트랜시버 자리 포함) |
